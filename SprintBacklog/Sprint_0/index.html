User Stories Break-down
  1. As Professor Ben, I want to add students to a course.
    1. Initialize a mySQL database with tables to store professors (id, name, password) and students (id, name, password)
      2. point: 1
      Dependent: None
    2. Implement API for professors to add students.
      3. point: 3
      Dependent: 1
    3. Design a UI for professors to interact with.
      4. point: 8
      Dependent: None
    4. Connect UI with API.
      5. point: 2
      Dependent: 1 and 3
  2. As Professor Ben, I want to create a unit. (A unit is defined as a chapter in the course and a problem set is a generated set of similar problems.)
    1. Add a unit table (id, name, release date) to the database.
      2. point: 1
      Dependent: None
    2. Implement API for professors to add units.
      3. point: 2
      Dependent: 1
    3. Extend UI to allow professors to add units.
      4. point: 4
      Dependent: None
    4. Connect UI with API.
      5. point: 2
      Dependent: 1 and 3
  3. As Professor Sophie, I want to add problem sets with a due-date to a unit.
    1. Add a problem set table (id, unit id, name, release date, end date)
      2. point: 1
      Dependent: None
  4. As Professor Sophie, I want the software to be able to take problem sets that I've created and release them to sstudents with only changed variables.
    1. Extend API to read a problem set file by searching. Query will be done via problem set Id and problem set name and return a list of all files that match.
      2. point: 1
      Dependent: None
    2. Implement variable replacement for files being stored. Variables in a file denoted by [$var(x-y)] will be replaced by a value between x and y when they are added to a unit.
      3. point: 3
      Dependent: 1
    3. Extend API to allow the release of problem set files by changing the release date in the sql tables.
      4. point: 1
      Dependent: None
    4. Extend UI to allow the change of variables to be done.
      5. point: 2
      Dependent: None
    5. Extend UI to support the querying of problem set files that have been added to the unit.
      6. point: 2
      Dependent: None
    6. Extend UI to allow the release of problem set files.
      7. point: 2
      Dependent: None
    7. Connect the UI and API to allow the release of problem sets.
      8. point: 2
      Dependent: 3 and 6
    8. Connect the UI and API to allow the added problem sets to have changed variables.
      9. point: 2
      Dependent: 2, 4, and 5
  5. As Professor Ben, I want to create several distinct versions of a problem-set to release to students.
    1. Extend UI to display more than 1 problem set per unit if possible based on queried results.
      2. point: 1
      Dependent: None
